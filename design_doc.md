## Problem Statement
Students, professionals, and entrepreneurs frequently miss deadlines and commitments because existing productivity tools rely on passive reminders that are easy to ignore. The goal is to build an AI-powered productivity companion that proactively assists users in planning, prioritizing, and completing tasks before deadlines are missed. The solution must move beyond traditional passive reminders and focus on helping users take meaningful action.

## What Makes This Cool
The core "Chaos to Focus" flow. A user dumps their raw, unstructured thoughts, commitments, and deadlines. The AI immediately parses them, auto-schedules them into a weekly calendar, and places them into an Eisenhower matrix. The user then starts a "Focused Workspace" where a native Web Speech API voice coach keeps them accountable, checks in on them to prevent procrastination, and plays ambient focus sounds.

## Constraints
- Must be built as a React SPA with Vite and Vanilla CSS.
- Styling must follow the ElevenLabs design system (EB Garamond, Inter, off-white canvas, ink typography, hairline borders, floating gradient orbs).
- Application must run entirely client-side, persisting data via `localStorage`.
- The user must be able to input their own Gemini API Key in a Settings panel.
- Deployed on Google Cloud for submission.

## Premises
1. Traditional reminders are passive and easy to ignore; true productivity companions must be interactive and proactive.
2. Users need task breakdown (subtasks) to overcome the initial friction of starting a daunting task.
3. AI-driven auto-scheduling helps users clear the cognitive load of planning their week.

## Approaches Considered
### Approach A: Modular Component-Based React Architecture
Features are separated into clean, modular files under `src/components/` (e.g. Navigation, AICoach, EisenhowerMatrix, FocusedWorkspace, SmartScheduler, HabitTracker, Settings). Centralized state is managed via React Context.
- Effort: S (CC: ~30 min)
- Risk: Low
- Pros: Keeps codebase organized, makes it much easier to isolate and debug bugs during the final hours of the hackathon.
- Cons: Requires managing component imports and setting up a shared state context.

### Approach B: Monolithic Single-File Architecture
Entire application logic inside `src/App.jsx` and styling in `src/index.css`.
- Effort: S (CC: ~15 min)
- Risk: Medium
- Pros: Easiest to edit quickly since all code lives in one file and no import paths are needed.
- Cons: Highly prone to file bloat, hard to debug layout issues, and difficult to navigate.

## Recommended Approach
Approach A (Modular Component-Based React Architecture) is recommended because it provides the best balance of maintainability, readability, and ease of debugging during the final hours of the hackathon.
